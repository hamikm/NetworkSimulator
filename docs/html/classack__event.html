Detailed Description

This event should be queued when a destination host wants to send an ACK (not just duplicate ACKs–any ACKS).

The event automatically queues another ack_event so that duplicate ACKs will be sent in the future if the destination doesn't get the correct FLOW packets. Pending ack_events should be cancelled when the destination gets the correct FLOW packet.
